# SCSK ふるさと納税 模擬サイト  
（寄付ページの情報設計に関する A/B テスト用サイト）

## 1. 概要（目的と位置づけ）
本リポジトリは、大学授業 **CsP（キャップストーンプロジェクト）** の一環として実施する **アンケート調査用の模擬Webサイト**です。SCSKのふるさと納税ページを参考にした静的HTMLモックを用い、**寄付ページにおける情報構成の違いが、利用者の理解度および寄付意欲に与える影響**を A/Bテスト形式で検証することを目的としています。  
※ 本サイトは調査・教育目的の模擬ページであり、実際の寄付受付は行いません。

---

## 2. 研究・調査の背景
近年、ふるさと納税をはじめとする寄付型プラットフォームでは、「プロジェクトの意義や背景をどのように伝えるか」が寄付行動に影響を与えると考えられています。一方で、情報を簡潔にまとめた方が理解しやすいのか、ストーリー性を付加した方が共感や納得感が高まるのか、といった点については明確な比較が難しい側面があります。本調査では、**ページの見た目やUIを揃えたまま情報構成のみを変化させる**ことで、寄付ページ設計における有効な情報提示方法を検討します。

---

## 3. 調査方法（A/B テスト設計）
### 比較条件
| 項目 | 条件A | 条件B |
| --- | --- | --- |
| 基本レイアウト | 同一 | 同一 |
| プロジェクト概要 | 表示 | 表示 |
| 背景ストーリー | 表示しない | 別ページへの導線あり |
| 主な違い | 情報を簡潔に提示 | 詳細な背景理解を促す |

- **条件A**: プロジェクト概要・ビジョン・資金使途など、必要最低限の情報のみを提示する構成  
- **条件B**: 条件Aと同一の構成に加え、プロジェクトの背景や想いを伝えるストーリーLPへの導線を設けた構成

### 評価観点（アンケート項目想定）
- ページの見やすさ
- 内容の理解しやすさ
- プロジェクトへの共感度
- 寄付したいと感じた度合い

---

## 4. サイト構成
- `index.html`: 調査用エントリーページ。条件A/Bの概要と注意喚起を表示し、各条件へ遷移。
- `project_overview_a.html`: 条件A用ページ。プロジェクト概要のみを提示。
- `project_overview_b.html`: 条件B用ページ。条件Aと同一構成にストーリーLPへの導線を追加。
- `project_story_lp.html`: 条件Bで使用するストーリー詳細ページ（現在はプレースホルダー）。
- `assets/`: 共通スタイル・スクリプト・画像・フォント。

---

## 5. 実装上の方針
- 静的HTMLのみで構成し、外部通信は行わない。
- 寄付・SNS・数値表示はすべてダミー。
- A/B条件の切替は URL クエリおよび `data-variant` 属性で制御。
- UI差分以外の影響を排除するため、レイアウト・配色・文量は可能な限り統一。

## 6. 動作確認方法
### ローカル環境
- `index.html` を直接ブラウザで開く。  
- もしくは簡易HTTPサーバーを使用:
  ```bash
  python -m http.server
  ```

### 条件別直接アクセス
- 条件A: `project_overview_a.html?variant=a`
- 条件B: `project_overview_b.html?variant=b`

### 公開ページ（確認用）
- GitHub Pages: `https://shutowwwatawata.github.io/donation-page-evaluation.github.io/`
